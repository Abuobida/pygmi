Calculate IGRF Corrected Data

This calculates IGRF corrected version of the magnetic data. It requires, as input, a magnetic dataset for an area as well as a digital elevation model. Since the IGRF correction relates to latitude and longitude, the input dataset is first converted to degrees, then corrected, and finally converted back to meters. For this reason, the input projection must also be specified on this dialog.

Options

Input Projection

  • Datum - Can be either WGS84 or Cape
  • Projection - Currently, projections of UTM and Transverse Mercator are supported
  • UTM Zone - This only applies to UTM. Changing this will automatically set the Central Meridian.
  • Latitude of Origin - Latitude from which meters start to be measured. This is normally 0.
  • Central Meridian - Longitude from which meters start to be measured. This is set automatically by UTM zone.
  • Scale Factor - Projection parameter for Transverse Mercator
  • False Easting - A value added to the easting calculated from the central meridian.
  • False Northing - A value added to the northing calculated from the latitude of origin.

IGRF Options

  • Sensor Clearance above ground - This is the height, in meters, of the sensor measuring the magentic field, above the ground.
  • Date - This is the survey date. The IGRF field changes depending on when the magnetic data was collected, so the survey date must be taken into account.
  • Digital Elevation Model - raster grid of the terrain in height above sea level.
  • Magnetic Data - raster grid of the magnetic data to be corrected.
