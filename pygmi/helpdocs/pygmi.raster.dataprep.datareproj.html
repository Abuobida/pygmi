Dataset Reprojection

This is a versatile routine allowing for the reprojecting of a dataset between two projections. EPSG codes can be used for maximum versatility, or one can manually enter in parameters relating to geodetic, transverse mercator and UTM.

EPSG codes are available online at either http://www.epsg-registry.org/, http://spatialreference.org/ref/epsg/ or in the gcs.csv and pcs.csv files in your python GDAL data directory.

Options

The options are the same for both input and output projections, and will only be explained once.

  • Use EPSG Code - allows for the use of an EPSG code to define the projection. Selecting this will grey out all other options.
  • Projection - Currently, projections of Geodetic (latitude and longitude), UTM and Transverse Mercator are supported
  • Datum - Can be either WGS84 or Cape
  • Latitude of Origin - Latitude from which meters start to be measured. This is normally 0.
  • Central Meridian - Longitude from which meters start to be measured. This is set automatically by UTM zone.
  • Scale Factor - Projection parameter for Transverse Mercator
  • False Easting - A value added to the easting calculated from the central meridian.
  • False Northing - A value added to the northing calculated from the latitude of origin.
  • UTM Zone - This only applies to UTM. Changing this will automatically set the Central Meridian.
